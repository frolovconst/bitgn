# Execution Plans

This section tracks planned and completed implementation work.

## Layout

- [`active/`](./active/README.md): work that is proposed or in progress
- [`completed/`](./completed/README.md): plans that have been executed and are kept for history
- [`tech-debt-tracker.md`](./tech-debt-tracker.md): durable debt items that are not tied to one active plan

## How to use this section

- create a short plan for substantial work before or during implementation
- move completed plans into `completed/` with a brief outcome summary
- keep long-lived debt out of random notes and in the tracker
